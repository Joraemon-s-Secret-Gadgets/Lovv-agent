## 1. 한눈 요약

- 3개 모델이 **핵심 축(의도·감정 / 맥락·제약 / 입력형태 / 수정 / 엣지 / 신규)**에서 거의 동일하게 수렴 → 발산 포화.
- **이미 V2가 받는 것**: 발견·힐링·즉흥·모호/모순/짧은 입력·장소교체·전면리셋·날씨대비·조회. (✅ 다수)
- **V1·V2 문서에 없던 새 축 6종**(아래 §3)이 다수 모델에서 반복 등장 — 특히 **제약/접근성**과 **신규 능력 신호**.
- **범위 밖(⛔)**: 숙소·식당·예약·실시간혼잡·내비연동·오프라인맵·사진/음성 입력 — "원하는 사용자 많다"는 **수요 신호**로만 기록.

## 4. V2 확장 우선순위 (스코프 확정 반영)

스코프 정리 후, V2는 **추천 품질**로 다시 수렴. (조회·안전·출력변환·접근성은 제외/Front/보류로 빠짐.)
