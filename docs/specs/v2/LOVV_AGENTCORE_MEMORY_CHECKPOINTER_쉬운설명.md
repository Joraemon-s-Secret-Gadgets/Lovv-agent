# 🧭 쉬운 설명서 — AgentCore Memory 체크포인터 전환

> 이 문서는 사람이 **눈으로 구조를 빠르게 이해**하기 위한 안내서입니다.
> 에이전트가 실제 구현 시 읽는 정식 계약은
> [`LOVV_AGENTCORE_MEMORY_CHECKPOINTER_SPEC.md`](./LOVV_AGENTCORE_MEMORY_CHECKPOINTER_SPEC.md)
> 입니다. 둘은 짝입니다 (쉬운 설명 ↔ 정식 계약).

---

## 1. 한 문장 요약

지금 우리 에이전트는 **매번 기억을 잃어버립니다.** 한 번 추천하고 끝나면 그걸로
끝이라, "후보 중에 골라보세요" 같은 **멀티턴 대화(이어서 진행)**를 못 합니다.
이걸 가능하게 하려고 **체크포인터(중간 저장 장치)**를 다는 것이 이번 작업입니다.

---

## 2. 비유로 이해하기

게임 세이브포인트를 떠올리면 됩니다.

- **지금**: 세이브 기능이 없는 게임. 죽으면 처음부터. (= 매 요청이 독립 실행)
- **바꾼 뒤**: 세이브포인트가 생긴 게임. 중간에 멈췄다가 **이어하기** 가능.
  (= 사용자가 후보 목적지를 고르는 동안 상태를 저장해 뒀다가 재개)

이 "세이브 파일"을 어디에 둘지가 핵심인데, 우리는 AWS의 **AgentCore Memory(단기)**
에 둡니다. 이미 세션 대화도 거기 저장되니, **세이브 파일도 같은 곳에 합치자**는 것이
이번 결정입니다.

---

## 3. 가장 중요한 원칙 한 줄

> **"잠깐 쓰는 건 합치고(통합), 오래 보관하는 건 따로 둔다(분리)."**

- **잠깐 쓰는 것** (세이브 파일 + 세션 대화) → AgentCore Memory에 **함께** 저장,
  약 7일 뒤 자동 삭제.
- **오래 보관하는 것** (사용자 개인화 프로파일, 가명처리 대상) → 우리가 직접 만든
  **별도 저장소(커스텀 2층 티어)**에 그대로 둠. (개인정보 통제권을 안 넘기려고)

---

## 4. 지금 구조 vs 바꾼 뒤 구조 (그림)

### 지금 (기억 없음)

```
사용자 요청
   │
   ▼
[진입점] ──▶ [하니스] ──▶ [그래프 compile()]  ← 세이브 기능 없음
                                  │
                          추천 만들고 끝
                                  │
            "후보 골라주세요" 가 필요해도 → 그냥 종료 (이어하기 불가)
```

### 바꾼 뒤 (기억 있음)

```
사용자 요청 (가명 ID 포함)
   │
   ▼
[진입점] ──▶ thread_id(세션) + actor_id(가명) 같이 전달
   │
   ▼
[하니스] ──▶ [그래프 compile(checkpointer=AgentCore Memory)]  ← 세이브 장착!
                                  │
                  ┌───────────────┴───────────────┐
            잠깐(세이브+세션)                 오래(개인화 프로파일)
          AgentCore Memory 7일                커스텀 별도 저장소
            └─ 이어하기 가능 ─┘
```

## 5. 어떤 파일을 손대나요? (사람이 보는 지도)

우리 코드는 `src/lovv_agent/` 안에 있습니다. 단계별로 나눠서 바꿉니다.

### 🟢 1단계 — 안전한 준비 작업 (동작은 그대로)

| 파일 | 무엇을 하나 | 쉬운 설명 |
|------|------------|-----------|
| `graph.py` | `compile()`에 세이브 슬롯 추가 | 세이브 **장착 구멍**만 뚫어둠. 안 꽂으면 지금과 똑같음 |
| `config.py` | Memory 설정 칸 추가 | "세이브 켤까 말까" **스위치**를 만듦 (기본 꺼짐) |
| `agentcore_entrypoint.py` | 가명 ID·세션 ID 전달 | 누가·어떤 세션인지 **이름표**를 그래프까지 들고 감 |
| `harness.py` | 위 부품들 연결 | 스위치 → 세이브 슬롯 **배선** (1단계엔 꺼둠) |

👉 **이 단계만 해도** 멈춰뒀다 이어하기 위한 "자리"가 다 생기고, **기존 동작은
전혀 안 바뀝니다.** AWS도 필요 없어 가장 안전합니다.

### 🟡 중간 관문 — 반드시 먼저 해결할 숙제

세이브 파일을 진짜로 디스크에 저장하려면, 우리 상태(state)를 **글자/숫자로
바꿨다가 다시 원래대로 복원**할 수 있어야 합니다. 지금은 "바꾸기"만 있고
"되돌리기"가 없어서, 이걸 먼저 만들어야 2단계로 갈 수 있습니다.

```
지금:  상태 ──(바꾸기 to_dict)──▶ 글자   ❌ 되돌리기 없음
필요:  상태 ◀─(되돌리기 from_dict)── 글자  ✅ 추가해야 함
```

### 🟣 2단계 — 실제 세이브 기능 켜기 (환경설정으로만)

| 파일 | 무엇을 하나 |
|------|------------|
| `pyproject.toml` | AWS 세이브 라이브러리 추가 (버전 고정) |
| `adapters/agentcore_memory.py` (새 파일) | 세이브 장치 **실제 생성** |
| `state.py` | 위 "되돌리기" 기능 추가 |
| `harness.py` | 설정이 켜지면 세이브 장치 연결 |

👉 기본은 **꺼짐**. `LOVV_MEMORY_ENABLED=true`로 켤 때만 동작. AWS 실제 자원이
있어야 완전 검증 가능 (로컬에선 일부만 확인).

### ⚪ 3단계 — 진짜 "이어하기" 대화 (나중에, 별도로)

"후보 골라주세요"에서 멈췄다가 사용자의 선택으로 **정확히 그 지점부터 재개**하는
기능. 그래프의 흐름 자체를 바꾸는 일이라 **위험이 커서 따로** 진행합니다.

---

## 6. 왜 한 번에 다 안 하나요?

두 가지 걸림돌 때문입니다.

1. **세이브 라이브러리가 아직 안 깔려 있음** → 2단계에서 추가 (버전이 자주 바뀌는
   프리뷰 단계라 조심).
2. **"되돌리기" 기능이 없음** → 이게 없으면 세이브 파일을 못 읽음. 그래서 중간
   관문을 먼저 통과해야 함.

그래서 **1단계(안전) → 중간 관문 → 2단계 → 3단계** 순서로 나눴습니다.
1단계만 해도 ADR에서 정한 "이름표 전달"과 "세이브 슬롯"이 확보됩니다.

---

## 7. 이름표(식별자) 규칙 — 꼭 기억할 것

| 우리말 | LangGraph | AgentCore | 들어가는 값 |
|--------|-----------|-----------|-------------|
| 세션 | `thread_id` | `session_id` | 어느 대화인지 |
| 사용자 | `actor_id` | `actor_id` | **가명 ID** (실명 아님!) |
| 보관기간 | — | 만료일수 | 약 7일 (이어하기 허용 최대 기간) |

⚠️ `actor_id`에는 **반드시 가명 ID**만 넣습니다. 실제 신원은 별도 매핑 테이블에만
있고, 이 코드는 거기 손대지 않습니다. (개인정보 보호 경계 유지)

---

## 8. 안전장치 요약

- 1단계는 **기존 테스트 전부 통과 + 동작 불변**이 목표 → 회귀 위험 거의 없음.
- 세이브는 **기본 꺼짐**, 명시적으로 켤 때만 작동.
- 개인정보(개인화 프로파일)는 **우리 저장소에 그대로** → 통제권 유지.
- 정식 SPEC의 요구사항(R1~R7)과 1:1로 연결됨.

---

## 9. 확인이 더 필요한 것 (솔직한 한계)

- AWS 세이브 라이브러리의 정확한 사용법은 **설치 후 확인** 필요 (프리뷰라 변동 가능) —
  신뢰도 중간.
- "되돌리기" 기능이 모든 중첩 데이터를 완벽히 복원하는지 **정밀 점검** 필요.
- 만료일수 최종값(7일?)은 실제 대화 길이를 보고 정함.
